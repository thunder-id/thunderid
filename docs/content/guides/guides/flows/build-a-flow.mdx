# Build a Sign-in Flow

By the end of this guide you will have a working username and password sign-in flow assigned to your application.

## Prerequisites

- <ProductName /> is running. See [Get Started](../../getting-started/get-thunderid).
- You have created an application. See [Quickstart](../../quick-start/quickstart).

<Stepper>

## Open the Flow Designer

1. Sign in to the <ProductName /> Console at [https://localhost:8090/console](https://localhost:8090/console).
2. Go to **Flows** → **+ Create New Flow**.
3. Give your flow a name — for example, `Sign In`.

The canvas opens with a default starter flow already wired: a **Sign In** view node → **Identifier + Password** executor → **Auth Assertion Generator** executor → **END**. This is exactly the flow you need for username and password authentication.

:::tip You're almost done
The default flow is a complete, working username and password sign-in flow. If that's all you need, skip to [Assign the flow to your application](#assign-the-flow-to-your-application).
:::

## Review the Canvas

The default flow has three nodes:

| Node | What it does |
|---|---|
| **Sign In** (View) | Renders the sign-in form with username and password fields |
| **Identifier + Password** (Executor) | Validates the submitted credentials against the user store |
| **Auth Assertion Generator** (Executor) | Issues the authentication assertion on success |

Each node connects to the next via its **green (success) output**. The final executor connects to **END**.

:::note
Every executor also has a **red (failure) output**. In the default flow this loops back to the Sign In view, so users see an error and can retry. Don't leave failure paths unconnected — an unconnected failure path stops the flow with an error.
:::

## Save the Flow

Click **Save** in the top-right corner. <ProductName /> saves the flow and assigns it a version number.

## Assign the Flow to Your Application

A flow only activates once assigned to an application.

1. In the Console, go to **Applications** and open the application you created in the Quickstart.
2. Open the **Flows** tab.
3. Under **Authentication Flow**, select the flow you just saved from the dropdown.
4. Click **Save**.

<ProductName /> will now run this flow whenever a user signs in to your application.

</Stepper>

## Go Further

Once you have the baseline working, the Flow Designer lets you add steps to your flow without changing your application code.

<details>
  <summary>Add multi-factor authentication (MFA)</summary>

1. From the <ProductName /> Console, go to **Applications** and open the application you want to configure.
2. Select the **Flows** tab.
3. Under **Authentication Flow**, **Registration Flow**, or **Recovery Flow**, choose the flow you created from the dropdown.
4. Click **Save**.

<ProductName /> executes the assigned flow when a user initiates the corresponding action (sign in, sign up, or password recovery).

:::note
For **Recovery Flow**, toggle the switch to enable password recovery before selecting a flow.
:::

</details>

<details>
  <summary>Add passkey authentication</summary>

  1. In the left panel under **Widgets**, click **+** next to **Passkey**.
  2. Wire the passkey executor between your Sign In view and the Auth Assertion Generator.
  3. To offer passkeys alongside password, connect the Sign In view to both executors and merge the success paths.
  4. Save and reassign to your application.

</details>

<details>
  <summary>Add social login</summary>

  1. Configure a social identity provider first — go to **Integrations** → **Identity Providers** and add Google, GitHub, or another provider.
  2. In the Flow Designer, add a **Social Login** executor from the left panel.
  3. Wire it in parallel with or as an alternative to the **Identifier + Password** executor.
  4. Save.

</details>

<details>
  <summary>Build from scratch</summary>

  To start with a blank canvas instead of the default flow:

  1. In the left panel, expand **Starter Templates**.
  2. Click **+** next to **Blank** to clear the canvas to a single START node.
  3. Add nodes from **Widgets** (pre-wired View + Executor pairs) or **Steps** (View nodes you wire manually).
  4. Connect nodes: drag from an output port to an input port.
  5. Every flow must end with an **Auth Assertion Generator** → **END** chain.

</details>

# Server Metadata

**Server Metadata** ([RFC 8414](https://datatracker.ietf.org/doc/html/rfc8414) and [OpenID Connect Discovery 1.0](https://openid.net/specs/openid-connect-discovery-1_0.html)) is a standardized JSON document that lists every OAuth 2.1 and OpenID Connect endpoint, every supported algorithm, and every feature flag for a <ProductName /> instance. Clients fetch it once at startup (or on demand) and use it instead of hard-coding endpoint URLs.

<ProductName /> publishes two documents — one for OAuth and one for OIDC. The OIDC document is a strict superset of the OAuth one.

## How It Works

| Endpoint | Spec |
|---|---|
| `GET /.well-known/oauth-authorization-server` | RFC 8414 |
| `GET /.well-known/openid-configuration` | OIDC Discovery 1.0 |

Both endpoints are public — no authentication required.

```bash
curl https://{{productSlug}}.example.com/.well-known/openid-configuration
```

The OAuth-only document at `/.well-known/oauth-authorization-server` omits the OIDC-specific fields (subject types, ID token algorithms, claims supported).

<details>
<summary>How <ProductName /> Implements It</summary>

| Aspect | Behavior |
|---|---|
| Caching | The document is generated per-instance from runtime configuration. Cache freely on the client. |
| `issuer` | The deployment's configured issuer URL. Must match the `iss` claim in issued tokens. |
| Endpoint URLs | Absolute URLs rooted at the configured issuer |
| Subject types | `public` only — pairwise subject identifiers are not supported |
| `require_pushed_authorization_requests` | Reflects the global `oauth.par.require_par` setting |
| `authorization_response_iss_parameter_supported` | Always `true` (see [Issuer Identification](../issuer-identification)) |

</details>

## Try It in <ProductName />

No configuration is needed. The discovery endpoints are always served and reflect the current deployment's settings.

### Use It from a Client

Most OAuth and OIDC SDKs accept a discovery URL and configure themselves from it:

```javascript
// Generic OIDC client (pseudocode)
const config = await fetch(
  'https://{{productSlug}}.example.com/.well-known/openid-configuration'
).then(r => r.json());

const client = new OIDCClient({
  authorization_endpoint: config.authorization_endpoint,
  token_endpoint: config.token_endpoint,
  jwks_uri: config.jwks_uri,
  // ...
});
```

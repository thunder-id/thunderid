## Run <ProductName />

Start a local <ProductName /> instance. Pick the method that works best for you:

<RunThunderID />

Once it's running, the console is available at [https://localhost:8090/console](https://localhost:8090/console).

## Create an Application

:::info Test User
If you used the default setup, sign in to the Console as `admin` with the password generated during setup and printed to the setup output (unless you supplied your own).
:::

Open the Console at <ConsoleUrl />, navigate to **Applications**, and click **Add Application**:

1. Under **Technology**, select **Next.js**.
2. Enter a name (e.g. `My Next.js App`) and create an application. The rest of the settings can stay at their defaults.
3. Copy both the **Client ID** and **Client Secret** from the window that pops up. The Client ID can also be found in the **General** tab.
4. Under **General**, add `http://localhost:3000` to the list of **Authorized Redirect URIs**.

## Create a Next.js App

Create your new Next.js app:

<CodeGroup>
  <CodeBlock lang="bash" label="npm">
    npx create-next-app@latest my-nextjs-app
    cd my-nextjs-app
  </CodeBlock>

  <CodeBlock lang="bash" label="Yarn">
    yarn create next-app my-nextjs-app
    cd my-nextjs-app
  </CodeBlock>

  <CodeBlock lang="bash" label="pnpm">
    pnpm create next-app my-nextjs-app
    cd my-nextjs-app
  </CodeBlock>
</CodeGroup>

When prompted, select the **App Router** option (the default).

## Set Environment Variables

Create a `.env.local` file in your project root with the following:

:::warning Configuration
Replace `<your-client-id>` and `<your-client-secret>` with the values from your <ProductName /> application.

Generate a random string for `THUNDERID_SECRET` (at least 32 characters):

```bash
openssl rand -base64 32
```
:::

```dotenv title=".env.local" showLineNumbers
NEXT_PUBLIC_THUNDERID_BASE_URL=https://localhost:8090
NEXT_PUBLIC_THUNDERID_CLIENT_ID=<your-client-id>
THUNDERID_CLIENT_SECRET=<your-client-secret>
THUNDERID_SECRET=<a-random-secret-for-session-signing>
# DANGER: Disables ALL TLS verification. Only for local development with self-signed certs. NEVER use in production.
NODE_TLS_REJECT_UNAUTHORIZED=0
```

## Add `ThunderIDProvider` to Your Layout

Wrap your root layout with `ThunderIDProvider` from the server export. This enables authentication across your entire app.

:::info
`ThunderIDProvider` handles the OAuth callback automatically, so no manual callback route is needed. Make sure the authorized redirect URL in your <ProductName /> application settings is set to `http://localhost:3000`.
:::

```tsx title="app/layout.tsx" {3,32} showLineNumbers
import type { Metadata } from "next";
import { Geist, Geist_Mono } from "next/font/google";
import { ThunderIDProvider } from '@thunderid/nextjs/server'
import "./globals.css";

const geistSans = Geist({
  variable: "--font-geist-sans",
  subsets: ["latin"],
});

const geistMono = Geist_Mono({
  variable: "--font-geist-mono",
  subsets: ["latin"],
});

export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html
      lang="en"
      className={`${geistSans.variable} ${geistMono.variable} h-full antialiased`}
    >
      <body className="min-h-full flex flex-col">
        <ThunderIDProvider>{children}</ThunderIDProvider>
      </body>
    </html>
  );
}
```

## Add the <ProductName /> Proxy

Create a `proxy.ts` file at your project root to proxy requests through <ProductName /> and protect routes:

```ts title="proxy.ts" showLineNumbers
import {
  thunderIDProxy,
  createRouteMatcher,
} from '@thunderid/nextjs/server'

const isProtectedRoute = createRouteMatcher([
  // Add the paths you want to protect, e.g. '/dashboard(.*)'
])

export default thunderIDProxy(async (thunderid, request) => {
  if (isProtectedRoute(request)) {
    await thunderid.protectRoute()
  }
})

export const config = {
  matcher: ['/((?!_next/static|_next/image|favicon.ico).*)'],
}
```

## Build with <ProductName /> components

Update your home page with <ProductName /> authentication components:

```tsx title="app/page.tsx" {1,6-11} showLineNumbers
import { SignedIn, UserDropdown, SignedOut, SignInButton } from "@thunderid/nextjs";

export default function Home() {
  return (
    <section className="flex flex-col items-center justify-center min-h-screen py-2">
      <SignedIn>
        <UserDropdown />
      </SignedIn>
      <SignedOut>
        <SignInButton>Sign In</SignInButton>
      </SignedOut>
    </section>
  );
}
```

Start the development server:

<CodeGroup>
  <CodeBlock lang="bash" label="npm">
    npm run dev
  </CodeBlock>

  <CodeBlock lang="bash" label="Yarn">
    yarn dev
  </CodeBlock>

  <CodeBlock lang="bash" label="pnpm">
    pnpm dev
  </CodeBlock>
</CodeGroup>

Visit your app at [http://localhost:3000](http://localhost:3000)

:::tip Success
You should see the sign-in button. Click it to be redirected to the <ProductName />-hosted sign-in page. Authenticate with the test user you created in step 2, then return to your app with the user dropdown displayed.
:::

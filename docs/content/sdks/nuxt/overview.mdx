# Nuxt SDK

The <ProductName /> Nuxt SDK (`@thunderid/nuxt`) is a Nuxt module that integrates <ProductName /> authentication into Nuxt 3 applications. It provides composables, auto-registered components, route middleware, server utilities, and a set of built-in API routes, all with full SSR hydration support.

## Installation

Install the <ProductName /> Nuxt SDK using your preferred package manager:

<CodeGroup>
  <CodeBlock lang="bash" label="npm">
    npm install @thunderid/nuxt
  </CodeBlock>

  <CodeBlock lang="bash" label="Yarn">
    yarn add @thunderid/nuxt
  </CodeBlock>

  <CodeBlock lang="bash" label="pnpm">
    pnpm add @thunderid/nuxt
  </CodeBlock>

  <CodeBlock lang="bash" label="Bun">
    bun add @thunderid/nuxt
  </CodeBlock>
</CodeGroup>

## Getting Started

Register the module and provide your application credentials in `nuxt.config.ts`:

```ts title="nuxt.config.ts" showLineNumbers
export default defineNuxtConfig({
  modules: ['@thunderid/nuxt'],
  thunderid: {
    clientId: '<your-client-id>',
    clientSecret: '<your-client-secret>',
    baseUrl: 'https://localhost:8090',
  },
})
```

Then wrap your root layout with `<ThunderIDRoot />`:

```vue title="app.vue" showLineNumbers
<template>
  <ThunderIDRoot>
    <NuxtPage />
  </ThunderIDRoot>
</template>
```

For full setup instructions, see the [Nuxt quickstart guide](../../../getting-started/connect-your-application/nuxt).

## Features

### Configuration

The SDK is configured via the `thunderid` key in `nuxt.config.ts`.

- **[`Module Configuration`](../apis/configuration/module-configuration)**: Full reference for all configuration options

### Composables

Auto-imported composables available in every component and page without explicit imports.

- **[`useThunderID()`](../apis/composables/use-thunderid)**: Access auth state and trigger sign-in, sign-out, and sign-up flows
- **[`useUser()`](../apis/composables/use-user)**: Access the authenticated user object

### Components

All components are auto-registered with the `ThunderID` prefix and do not require explicit imports.

#### Provider
- **[`<ThunderIDRoot />`](../apis/components/thunderid-root)**: Root provider that mounts all authentication context providers

#### Control Components
- **[`<ThunderIDSignedIn />`](../apis/components/signed-in)**: Render content only when the user is authenticated
- **[`<ThunderIDSignedOut />`](../apis/components/signed-out)**: Render content only when the user is not authenticated
- **[`<ThunderIDLoading />`](../apis/components/loading)**: Render content during authentication loading state

#### Action Components
- **[`<ThunderIDSignInButton />`](../apis/components/sign-in-button)**: Trigger the sign-in flow
- **[`<ThunderIDSignOutButton />`](../apis/components/sign-out-button)**: Sign out the current user
- **[`<ThunderIDSignUpButton />`](../apis/components/sign-up-button)**: Trigger the sign-up flow

#### User Self-care Components
- **[`<ThunderIDUser />`](../apis/components/user)**: Access authenticated user data via slot props
- **[`<ThunderIDUserProfile />`](../apis/components/user-profile)**: Full profile management interface
- **[`<ThunderIDUserDropdown />`](../apis/components/user-dropdown)**: User menu with profile and sign-out options

### Middleware

- **[`defineThunderIDMiddleware()`](../apis/middleware/define-thunderid-middleware)**: Factory for creating route middleware that gates pages by auth state, organization, or required scopes
- Named **`'auth'`** middleware: pre-built middleware that redirects unauthenticated users to sign-in

### Server Utilities

Server-only utilities imported from `@thunderid/nuxt/server` for use in API routes, server routes, and server middleware.

- **[`useServerSession()`](../apis/server/use-server-session)**: Read the session payload from the current request, or `null`
- **[`requireServerSession()`](../apis/server/require-server-session)**: Read the session or throw a `401` error
- **[`getValidAccessToken()`](../apis/server/get-valid-access-token)**: Get a valid access token, refreshing it automatically if expired
- **[`get<ProductName />Context()`](../apis/server/get-thunderid-context)**: Read the typed <ProductName /> context attached to an H3 event

### Utilities

- **[`createRouteMatcher()`](../apis/utils/create-route-matcher)**: Build a route-matching function from glob patterns for use in middleware

### Error Handling

- **[`ThunderIDError`](../apis/errors/thunderid-error)**: Structured error class thrown by the SDK with typed `ErrorCode` values

## Built-in API Routes

The module automatically registers the following server routes under `/api/auth`:

| Route | Method | Description |
| :--- | :--- | :--- |
| `/api/auth/signin` | `GET` | Start redirect-based sign-in |
| `/api/auth/signin` | `POST` | Handle embedded sign-in step |
| `/api/auth/signup` | `POST` | Handle embedded sign-up step |
| `/api/auth/callback` | `GET / POST` | OAuth callback handler |
| `/api/auth/signout` | `POST` | Sign out and return redirect URL |
| `/api/auth/session` | `GET` | Get current auth state |
| `/api/auth/token` | `GET` | Get client-safe access token |
| `/api/auth/user` | `GET` | Get user object |
| `/api/auth/user/profile` | `GET / PATCH` | Get or update user profile |
| `/api/auth/branding` | `GET` | Get tenant branding preference |

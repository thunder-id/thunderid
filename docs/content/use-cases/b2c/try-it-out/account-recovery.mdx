# Account Recovery

In this walkthrough, John has forgotten his Wayfinder password. He selects **Forgot password?** on the sign-in screen, receives a recovery email, opens the link, and sets a new password. <ProductName /> runs the recovery flow end-to-end and returns him to Wayfinder ready to sign in with the new credentials.

:::note Prerequisites
Complete [Set Up Your Environment](../../try-it-out#set-up-your-environment) before starting this walkthrough.
:::

:::info Background
[Add Account Recovery](../../customer-identity#add-account-recovery) covers the requirements story behind this use case.
:::

## Pick Your Pattern

<details open>
  <summary>Redirect-based</summary>

In the redirect-based pattern, the consumer app sends the user to <ProductName /> for the entire recovery experience. The Wayfinder web frontend is configured exactly this way: the **Sign in** button triggers an OIDC redirect to <ProductName />. From there, the user picks **Forgot password?**, completes the recovery flow, and returns to Wayfinder to sign in with the new password.

**Try the Use Case**

1. Open <WayFinderSampleUrl /> and select **Sign in**. The browser navigates to <ProductName />.
2. On the <ProductName /> sign-in page, select **Forgot password?**.
3. Enter `john.doe` as the username and submit. <ProductName /> runs the recovery flow and sends a recovery email to `john.doe@example.com`.
4. Open the recovery email from the inbox at <WayFinderMailUrl /> and open the link inside.
5. <ProductName /> renders a **Set new password** screen. Enter a new password and submit.
6. <ProductName /> shows a confirmation and returns John to Wayfinder. Select **Sign in** again and sign in with the new password.

</details>

# Overview

Documentation improvements are fast-track contributions: no design proposal or maintainer assignment required before you open a PR.

You can browse open documentation issues on GitHub using the <RepoLink path="/issues?q=is%3Aissue%20state%3Aopen%20label%3AType%2FDocs">`Type/Docs`</RepoLink> label.

## Fast Track Process

1. **Find what needs improving**: typo, outdated info, missing example, unclear explanation
2. **Fork the repository** on GitHub
3. **Edit the file** under `docs/content/`
4. **Open a pull request** with a short description of what you changed and why

That's it. Docs PRs are reviewed quickly and are a great first contribution.

## Types of Improvements

- Fix typos or grammatical errors
- Clarify confusing explanations
- Add code examples or usage snippets
- Update screenshots or diagrams that are out of date
- Fill in missing sections or stub pages

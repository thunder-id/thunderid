# Vue SDK

The <ProductName /> Vue SDK (`@thunderid/vue`) provides a comprehensive set of components, composables, and utilities to integrate <ProductName /> authentication and user management into your Vue 3 applications.

## Installation

Install the <ProductName /> Vue SDK using your preferred package manager:

<CodeGroup>
  <CodeBlock lang="bash" label="npm">
    npm install @thunderid/vue
  </CodeBlock>

  <CodeBlock lang="bash" label="Yarn">
    yarn add @thunderid/vue
  </CodeBlock>

  <CodeBlock lang="bash" label="pnpm">
    pnpm add @thunderid/vue
  </CodeBlock>

  <CodeBlock lang="bash" label="Bun">
    bun add @thunderid/vue
  </CodeBlock>
</CodeGroup>

## Getting Started

To get started quickly, check out our [Vue Quickstart Guide](../../../guides/getting-started/connect-your-application/vue) for step-by-step instructions.

:::warning Use the redirect-based sign-in flow
Browser SPAs should sign in using the redirect-based OAuth2 `authorization_code` + PKCE flow. Use
the [`<SignInButton />`](../apis/components/sign-in-button) component.

**Initiating a sign-in flow directly in the browser**, the embedded `<SignIn />` component or
`useThunderID().signIn({ applicationId })` used standalone, is **not supported** for browser SPAs.
See
[Register an application](../../../guides/guides/applications/manage-applications).
:::

## Features

### Providers

The SDK provides a root provider component that manages authentication state across your application via Vue's `provide`/`inject`.

- **[`<ThunderIDProvider />`](../apis/providers/thunderid-provider)** - Root provider component that wraps your application and manages authentication state

### Components

Pre-built, customizable components for common authentication flows and user management.

#### Action Components
- **[`<SignInButton />`](../apis/components/sign-in-button)** - Trigger sign-in flow
- **[`<SignOutButton />`](../apis/components/sign-out-button)** - Sign out the current user
- **[`<SignUpButton />`](../apis/components/sign-up-button)** - Trigger sign-up flow

#### Control Components
- **[`<SignedIn />`](../apis/components/signed-in)** - Conditionally render content for authenticated users
- **[`<SignedOut />`](../apis/components/signed-out)** - Conditionally render content for unauthenticated users
- **[`<Loading />`](../apis/components/loading)** - Display loading state during authentication

#### User Self-care Components
- **[`<UserDropdown />`](../apis/components/user-dropdown)** - User menu with profile and settings
- **[`<UserProfile />`](../apis/components/user-profile)** - Complete user profile management interface
- **[`<User />`](../apis/components/user)** - Display user information

### Composables

Vue composables for programmatic access to authentication and user data.

- **[`useThunderID()`](../apis/composables/use-thunderid)** - Access the main <ProductName /> SDK instance

## Customization

All components in the <ProductName /> Vue SDK are fully customizable. You can:

- Override default styles using CSS classes or inline styles
- Customize component behavior through props
- Build custom components using the provided composables
- Use scoped slots to control rendering

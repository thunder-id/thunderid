# Quickstart

By the end of this guide you will have:
- A registered application in ThunderID with a **Client ID**
- A test user you can sign in with
- Everything ready to [build your sign-in flow](../guides/flows/build-a-flow) and [connect your app](connect-your-application/react)

## Prerequisites

- ThunderID is running. If not, [get ThunderID first](../getting-started/get-thunderid).
- You can reach the ThunderID Console at [https://localhost:8090/console](https://localhost:8090/console).

<Stepper>

## Create an application

An application in ThunderID represents your product. When you register one, you get a **Client ID** — this is what links your frontend or backend to ThunderID's authentication layer.

1. Sign in to the Console at [https://localhost:8090/console](https://localhost:8090/console) using the default admin credentials (`admin` / `admin`).
2. Go to **Applications** → **New Application**.
3. Enter an **Application Name** (for example, `my-app`).
4. Choose your **Sign-In Options** — the methods users will use to authenticate (for example, Username & Password).
5. Under **Sign-In Experience**, set the **Sign-In Approach**:
   - Choose **Redirect to ThunderID sign-in pages** — this is the recommended approach and the one this guide follows. Users are sent to ThunderID-hosted pages you control with the Flow Designer.
6. Under **User Access**, select **Customer** (for self-service end users) or **Person** (for admin-managed accounts). Select both if unsure.
7. Select your **Application Type**:

   | Type | Use when |
   |---|---|
   | **Browser App** | React, Angular, Vue, or any SPA running in the browser |
   | **Full-stack App** | Server-rendered apps (Next.js, Express, Spring) where the backend handles token exchange |
   | **Mobile App** | Native or cross-platform mobile (React Native, Flutter, Swift, Kotlin) |
   | **Backend Service** | Server-to-server calls with no user sign-in |

   For the React quickstart, choose **Browser App** and set the **Authorized Redirect URL** to `http://localhost:5173`.

8. Click **Create Application**.

:::tip Save your Client ID
Open the application details page and note the **Client ID**. You will need it when configuring the React SDK.
:::

## Create a Test User

You need at least one user to verify that sign-in works end-to-end.

1. In the Console, go to **Users** → **Add User**.
2. Select a **User Type** — choose **Customer** to match the access setting above.
3. Fill in the user details: `username`, `email`, `given_name`, `family_name`, and `password`.
4. Click **Create User**.

</Stepper>
